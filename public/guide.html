사전 FMEA 사용 설명서

다양한 차종·아이템의 SW FMEA를 자동화하는 도구입니다. 설계사양서·DBC·아키텍처 문서를 업로드하면 AI가 HAZOP 9가지 가이드워드를 적용하여 FMEA 항목을 자동으로 작성하고, AP·SPF/LF 분류까지 산출합니다.

세션 만들기

모든 작업은 세션 단위로 관리됩니다. 하나의 세션 = 하나의 FMEA 작업 단위입니다. 차종과 아이템이 다르면 별도 세션을 생성합니다.

새 세션 생성 방법
  1. 우상단 + 새 세션 만들기 버튼 클릭
  2. 아이템명 입력 — 분석할 SW 모듈명 (예: SBW, EPS, PLBM)
  3. 세션명 입력 — 차종_시스템명_버전 형태 권장 (예: GN7_EPS_사전FMEA_v1)
  4. 세션 만들기 클릭 → 1단계로 자동 진입

💡 세션 목록 카드에서 아이템별로 그룹화되어 표시되며, 각 세션의 생성 항목 수와 진행 상태를 바로 확인할 수 있습니다.

생성완료1단계에서 FMEA 항목이 생성된 상태

Step 2 — FMEA 자동 작성

아래 세 가지 조건이 모두 충족되면 버튼이 활성화됩니다.

활성화 조건
  • ✅ FMEA 양식 업로드 완료
  • ✅ SW 설계사양서 1개 이상 업로드 완료
  • ✅ 인터페이스 변수 추출 완료 (Step 1)

추출된 인터페이스 변수 × HAZOP 9가이드워드 조합으로 항목을 생성합니다. 30~120초 소요됩니다.

이미 항목이 있을 경우 버튼이 "🔄 재실행 — 기존 N개 대체"로 표시됩니다.

⚠ 재실행 시 기존 항목이 모두 삭제되고 새로 생성됩니다. 2단계에서 작업한 내용도 초기화되므로 주의하세요.

3단계 — 피드백 & 고도화

1단계에서 FMEA 항목이 생성된 후 진입 가능합니다. 2단계 리포트를 확인한 뒤 항목의 품질을 검토하고 전문가 검토본과 병합합니다.

VH 항목 집중 검토

AP=VH인 항목만 필터하여 빨간 테두리 테이블로 표시합니다. VH 항목이 없으면 섹션이 숨겨집니다.

확인 포인트
  • S·O·D 값이 실제 시스템 위험도와 일치하는지 검토
  • 예방 조치·검출 조치 내용이 구체적으로 기재되어 있는지 확인
  • Counter Measure 상세 내용은 2단계 리포트 ⑤에서 확인

Gap 분석 선택사항

전문가가 검토·완성한 FMEA Excel 파일을 업로드하면 AI 자동 생성본과 대조하여 SOD 오차·누락 항목을 검출합니다. 전문가 검토본이 없는 경우 이 단계는 건너뛸 수 있습니다.

Gap 유형의미
누락AI가 생성하지 못한 항목 (전문가 검토본에만 존재)
SOD_S/O/DS·O·D 값이 전문가 기준과 2 이상 차이 나는 항목
조치부족예방/검출 조치 내용이 전문가 기준보다 불충분한 항목

고도화 (AI + 전문가 병합)

Gap 분석 결과를 바탕으로 전문가 검토본의 SOD 값을 우선 적용하고 누락 항목을 보완합니다. 전문가 검증을 통과하지 못한 항목은 신뢰도가 낮아집니다.

💡 Gap 분석 없이도 세션 상태가 검토완료 또는 고도화완료이면 고도화 실행이 가능합니다.

통계 카드의미
🔴 AP=VH 항목현재 즉시 조치가 필요한 항목 수
✅ 검토 확정review_status = accepted 또는 modified 항목 수
⏳ 검토 대기review_status = pending 항목 수

AP 재계산

현재 S·O·D 값을 기준으로 AIAG-VDA 룩업 테이블에 따라 모든 항목의 AP를 다시 계산합니다. 항목의 SOD 값을 수정했거나 Counter Measure 적용 후 S*/O*/D* After가 변경된 경우 실행하세요.

⚠ 재계산 후 2단계 리포트를 재생성하면 AP* After 분포가 정확하게 반영됩니다.

재계산 완료 후 섹션 하단에 현재 AP 분포 (VH/H/M/L 각 개수)가 실시간으로 표시됩니다.

HAZOP 가이드워드

9가지 가이드워드를 인터페이스 변수 각각에 적용하여 고장 형태를 도출합니다.

MORE
신호값이 정상보다 크게 출력
LESS
신호값이 정상보다 작게 출력
NO
신호가 전혀 출력되지 않음
REVERSE
신호 방향 또는 극성이 반전
CORRUPT
신호값이 손상 또는 오염
EARLY
신호가 예정보다 일찍 발생
LATE
신호가 예정보다 늦게 발생
AS_WELL_AS
정상 신호 외 추가 신호 발생
PART_OF
신호의 일부만 전달

주요 개념

  • SPF Single Point Failure. Safety Mechanism이 없으면 직접 Safety Goal 위반으로 이어지는 고장.
  • LF Latent Failure. 단독으로는 Safety Goal을 위반하지 않지만, 다른 고장과 결합 시 위험해지는 고장.
  • Safety Mechanism 고장의 영향을 감지·제어·완화하는 기술적 수단. Preventive(예방)와 Detection(감지) 두 종류로 구분.
  • D2 Safety Mechanism의 Diagnostic Coverage(진단 포함률)를 반영한 보정 Detection 값. D2가 낮을수록 SM이 효과적으로 적용된 것.
  • Counter Measure AP=VH 또는 고위험 항목에 대해 제안되는 설계 개선 조치. 적용 시 S*/O*/D* After 값과 AP* After가 개선됨.
  • ICD Interface Control Document. SW 컴포넌트 간 인터페이스 변수 목록. 아키텍처+DBC 파일로부터 자동 추출.
  • 인터페이스 변수 SW 컴포넌트 간 또는 외부와 주고받는 신호·데이터. FMEA에서 각 변수마다 9가지 HAZOP 가이드워드를 적용하여 고장 형태를 도출함.
